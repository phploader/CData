# **1. Core Concepts of CData**

CData is based on a simple but very powerful principle:
**All data ŌĆō both reading and writing ŌĆō is controlled through two arrays: `$D` and `$F`.**
These two structures are the heart of CData and allow complex data models to be mapped without SQL and without an ORM.

---

# **$D ŌĆō The Data Container**

`$D` is a **multidimensional array** that contains both input data (for `set_object()`) and output data (from `get_object()`).
The structure is **always identical**, whether data is being read or written.

Example for an article:

```php
$D['ARTICLE']['D']['article-id']['Active'] = 1;
$D['ARTICLE']['D']['article-id']['Name']   = 'My Article';
```

### Properties of `$D`:

- Each table is its own key (`ARTICLE`, `ATTRIBUTE`, `STOCK`, ŌĆ”)
- Records are stored under `D`
- The **record ID is always the array key**
- IDs can contain numbers and letters
- CHILD tables are nested recursively under `D`
- The structure can be arbitrarily deep

Example for a deep structure:

```
ARTICLE
  D
    123
      ATTRIBUTE
        D
          Color
            LANGUAGE
              DE
                Title
```

---

# **$F ŌĆō The Filter Container**

`$F` is used exclusively for `get_object()` and determines **which data should be loaded**.

Example: load all articles

```php
$F['ARTICLE'] = [];
```

### WHERE filters (`W`)

- `W[0]` = AND block
- `W[1]` = OR block

Example: active articles with name "Article A"

```php
$F['ARTICLE']['W'][0]['Active'] = 1;
$F['ARTICLE']['W'][0]['Name']   = 'Article A';
```

Example: also articles with name "Article B" (OR)

```php
$F['ARTICLE']['W'][1]['Name'] = 'Article B';
```

### Comparison operators

```php
$F['ARTICLE']['W'][0]['Price']['>']  = 10;
$F['ARTICLE']['W'][0]['Price']['<']  = 100;
$F['ARTICLE']['W'][0]['Active']['!='] = 0;
```

### LIKE variants

```php
$F['ARTICLE']['W'][0]['Name']['LIKE']   = '%test%';
$F['ARTICLE']['W'][0]['Name']['LIKE%%'] = 'test';
$F['ARTICLE']['W'][0]['Name']['LIKE%-'] = 'test';
$F['ARTICLE']['W'][0]['Name']['LIKE-%'] = 'test';
```

### IN / NOT IN

```php
$F['ARTICLE']['W'][0]['ID'] = ['1','2','3'];          // IN
$F['ARTICLE']['W'][0]['ID']['NOTIN'] = ['1','2','3']; // NOT IN
```

---

# **ORDER & LIMIT**

Order, limit, paging, and sorting are defined through `$F`.

```php
$F['ARTICLE']['O'][0]['Name'] = 'ASC';
$F['ARTICLE']['L']['STEP']  = 10;
$F['ARTICLE']['L']['START'] = 0;
```

---

# **Pattern ŌĆō The Structure Definition**

CData creates tables automatically based on a pattern array:

```php
$D['PATTERN']['ARTICLE'] = [
    'Active'   => ['Type' => 'checkbox'],
    'Name'     => ['Type' => 'text'],
    'ParentId' => ['Type' => 'id', 'ForeignKey' => 1],
];
```

CHILD tables are defined via `D`:

```php
$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE'] = [
    'Active' => ['Type' => 'checkbox'],
    'Name'   => ['Type' => 'text'],
];
```

---

# **get_object() ŌĆō Reading Data**

```php
$db->get_object($D, $F);
```

- `$F` determines what is loaded
- `$D` is filled with the results
- CHILD tables are loaded recursively

---

# **Summary**

- `$D` is the data container for reading and writing
- `$F` is the filter container for queries
- The structure is defined by a pattern
- `get_object()` and `set_object()` work with the same recursive structure
