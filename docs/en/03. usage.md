# **3. Reading Data – get_object()**

The method `get_object()` is the central mechanism for loading data from the database.
It works exclusively with the **filter array `$F`** and writes the results into the **data array `$D`**.
The structure of `$D` exactly matches the structure used for saving data (`set_object()`).

---

# **Basic Principle**

```php
$db->get_object($D, $F);
```

- `$F` determines **what** is loaded
- `$D` is filled with the loaded data
- the structure is recursive and follows the pattern
- CHILD tables are loaded automatically when included in the filter

---

# **3.1 Loading Tables**

To load a table, an empty entry is enough:

```php
$F['ARTICLE'] = [];
$db->get_object($D, $F);
```

Result:

```php
$D['ARTICLE']['D'][ID][fieldname]
```

---

# **3.2 Loading Records by ID**

```php
$F['ARTICLE']['W'][0]['ID'] = '123';
```

Multiple IDs:

```php
$F['ARTICLE']['W'][0]['ID'] = ['123', '456'];
```

---

# **3.3 WHERE Filters (W)**

### AND filter (`W[0]`)

```php
$F['ARTICLE']['W'][0]['Active'] = 1;
$F['ARTICLE']['W'][0]['Name']   = 'Article A';
```

### OR filter (`W[1]`)

```php
$F['ARTICLE']['W'][1]['Name'] = 'Article B';
```

---

# **3.4 Comparison Operators**

```php
$F['ARTICLE']['W'][0]['Price']['>']  = 10;
$F['ARTICLE']['W'][0]['Price']['<']  = 100;
$F['ARTICLE']['W'][0]['Active']['!='] = 0;
```

---

# **3.5 LIKE Variants**

CData supports multiple LIKE forms:

```php
$F['ARTICLE']['W'][0]['Name']['LIKE']   = '%test%';   // contains
$F['ARTICLE']['W'][0]['Name']['LIKE%%'] = 'test';     // contains (short form)
$F['ARTICLE']['W'][0]['Name']['LIKE%-'] = 'test';     // ends with
$F['ARTICLE']['W'][0]['Name']['LIKE-%'] = 'test';     // starts with
```

---

# **3.6 IN / NOT IN**

```php
$F['ARTICLE']['W'][0]['ID'] = ['1','2','3'];          // IN
$F['ARTICLE']['W'][0]['ID']['NOTIN'] = ['1','2','3']; // NOT IN
```

---

# **3.7 Sorting – ORDER (O)**

```php
$F['ARTICLE']['O'][0]['Name'] = 'ASC';
```

Multiple sorts:

```php
$F['ARTICLE']['O'][0]['Active'] = 'DESC';
$F['ARTICLE']['O'][1]['Name']   = 'ASC';
```

---

# **3.8 Limit & Offset – LIMIT (L)**

```php
$F['ARTICLE']['L']['STEP']  = 10; // number of records
$F['ARTICLE']['L']['START'] = 0;  // offset
```

---

# **3.9 Loading CHILD Tables**

CHILD tables are recognized via the pattern and can be loaded selectively.

### Load all articles + all attributes

```php
$F['ARTICLE']['ATTRIBUTE'] = [];
```

### Filter articles by attributes

```php
$F['ARTICLE']['W'][0]['ATTRIBUTE']['W'][0]['Name'] = 'Hello';
```

→ Only articles with an attribute "Hello" are returned

### Load all articles, but only certain attributes

```php
$F['ARTICLE']['ATTRIBUTE']['W'][0]['Name'] = 'Hello';
```

→ Articles are **not** filtered, only the attributes are

---

# **3.10 Loading Multiple Tables Simultaneously**

```php
$F['ARTICLE']  = [];
$F['ATTRIBUTE'] = [];
$F['STOCK']     = [];

$db->get_object($D, $F);
```

---

# **3.11 Result Structure in `$D`**

After loading, `$D` might look like this:

```php
$D['ARTICLE']['D']['123']['Active'] = 1;
$D['ARTICLE']['D']['123']['Name']   = 'Test Article';

$D['ARTICLE']['D']['123']['ATTRIBUTE']['D']['Color']['Name'] = 'Red';
```

The structure matches the pattern exactly.

---

# **Summary**

- `get_object()` loads data recursively based on `$F`
- `$D` contains the complete data structure
- filters support AND, OR, LIKE, IN, NOT IN, and operators
- CHILD tables can be filtered or loaded separately
- sorting and limit work at every level
- the structure can be arbitrarily deep
