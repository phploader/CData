# **Welcome to the CData documentation**

CData is a lightweight yet highly flexible data management system for PHP.  
It allows you to work with complex, recursive data structures—without SQL queries, ORM models, or rigid database schemas. Instead, CData is entirely based on two central concepts:

- **`$D`** – the data container for reading and writing  
- **`$F`** – the filter container for queries  

The entire database structure is defined via a **pattern system** that describes tables, fields, relationships, and CHILD structures.  
From this, CData automatically generates the required SQLite tables and manages insert, update, and delete operations on its own.

This wiki complements the README and provides detailed, practice‑oriented documentation with many examples, best practices, and explanations of the internal workings.

## **Who is CData for?**

CData is aimed at developers who:

- need flexible data structures  
- want to work without SQL  
- want to model complex relationships without ORM overhead  
- prefer a system that manages data and structure in a single array  
- develop modular, extensible applications  

Whether it’s a shop system, CMS, configurator, or data management tool—CData is suitable for all projects that require dynamic structures.
