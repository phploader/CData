# **2. Pattern Definition**

The pattern system is the foundation of CData.
It describes the complete database structure – tables, fields, data types, relationships, and recursive CHILD structures.
CData automatically generates SQLite tables based on the pattern and manages all data operations (insert, update, delete).

A pattern is defined once and then registered with:

```php
$C['CData']->registerPattern($D['PATTERN']);
```

---

# **What is a Pattern?**

A pattern is an array that describes:

- which tables exist
- which fields these tables have
- which data types the fields use
- which CHILD tables belong to them
- how deep the structure goes (arbitrarily deep)

Example for a simple table:

```php
$D['PATTERN']['ARTICLE'] = [
    'Active'   => ['Type' => 'checkbox'],
    'Name'     => ['Type' => 'text'],
    'ParentId' => ['Type' => 'id', 'ForeignKey' => 1],
];
```

This creates an `ARTICLE` table with the fields:

- Active
- Name
- ParentId

---

# **Supported Data Types**

| Type | Description |
|------|-------------|
| checkbox | boolean value (0/1) |
| text | plain text |
| id | alphanumeric ID |
| number | numeric value |
| json | JSON value |
| date | date |

> New types can be added at any time by extending the CData class.

---

# **Defining CHILD Tables**

CHILD tables are defined via the `D` key.
This allows recursive structures of arbitrary depth.

Example: articles with attributes

```php
$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE'] = [
    'Active' => ['Type' => 'checkbox'],
    'Name'   => ['Type' => 'text'],
];
```

This creates a structure:

```
ARTICLE
  ATTRIBUTE
```

CData automatically creates the necessary tables and relationships.

---

# **Multi-level CHILD Structures**

CData supports arbitrarily deep structures.

Example:

```php
$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE']['D']['LANGUAGE'] = [
    'Title' => ['Type' => 'text'],
];
```

Structure:

```
ARTICLE
  ATTRIBUTE
    LANGUAGE
```

This structure can be extended arbitrarily:

```
ARTICLE
  ATTRIBUTE
    PROPERTY
      LANGUAGE
        META
```

There is **no depth limit**.

---

# **Foreign Keys**

A field can be marked as a foreign key:

```php
'ParentId' => ['Type' => 'id', 'ForeignKey' => 1],
```

This is used to model relationships.
CData creates an internal structure such as:

```
$D['ARTICLE']['ForeignKey']['Kind-ids']
```

Foreign keys are **not** defined via SQL constraints, but through CData's internal structure management.

---

# **Example: Complex Pattern**

```php
$D['PATTERN']['ARTICLE'] = [
    'Active'   => ['Type' => 'checkbox'],
    'Name'     => ['Type' => 'text'],
    'ParentId' => ['Type' => 'id', 'ForeignKey' => 1],
];

$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE'] = [
    'Active' => ['Type' => 'checkbox'],
    'Name'   => ['Type' => 'text'],
];

$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE']['D']['LANGUAGE'] = [
    'Title' => ['Type' => 'text'],
];
```

Resulting structure:

```
ARTICLE
  ATTRIBUTE
    LANGUAGE
```

---

# **Registering a Pattern**

After defining the pattern, it must be registered:

```php
$C['CData']->registerPattern($D['PATTERN']);
```

CData then automatically creates:

- tables
- fields
- CHILD tables
- internal relationships
- storage logic

---

# **Summary**

- A pattern defines the complete database structure.
- Tables and fields are created automatically.
- CHILD tables are defined recursively via `D`.
- The structure can be arbitrarily deep.
- Data types govern storage and processing.
- Foreign keys are used for internal structure management.

The pattern is the foundation for all operations in CData and determines how data is read, stored, and structured.
