# **4. Filterlogik**

Die Filterlogik von CData ist eines der mächtigsten Elemente des Systems.\
Sie ermöglicht es, Daten präzise und flexibel aus der Datenbank zu laden – ohne SQL schreiben zu müssen.\
Alle Filter werden im Array **`$F`** definiert und an `get_object()` übergeben.

```php
$db->get_object($D, $F);
```

---

# **4.1 Grundstruktur eines Filters**

Ein Filter besteht immer aus:

```
$F['TABELLE']['W'][Block][Feld] = Wert;
```

- **TABELLE** = Name der Tabelle aus dem Pattern
- **W** = WHERE‑Bedingungen
- **Block** = 0, 1, 2 … (0 = AND, 1 = OR)
- **Feld** = Spaltenname oder CHILD‑Struktur
- **Wert** = Vergleichswert oder Operator

Beispiel:

```php
$F['ARTICLE']['W'][0]['Active'] = 1;
```

---

# **4.2 AND‑Bedingungen (W\[0\])**

Alle Bedingungen in `W[0]` werden mit **UND** verknüpft.

```php
$F['ARTICLE']['W'][0]['Active'] = 1;
$F['ARTICLE']['W'][0]['Name']   = 'Artikel A';
```

Entspricht:

```
WHERE Active = 1 AND Name = 'Artikel A'
```

---

# **4.3 OR‑Bedingungen (W\[1\])**

Jeder zusätzliche Block (`W[1]`, `W[2]`, …) ist ein **ODER‑Block**.

```php
$F['ARTICLE']['W'][1]['Name'] = 'Artikel B';
```

Entspricht:

```
OR Name = 'Artikel B'
```

---

# **4.4 Vergleichsoperatoren**

CData unterstützt verschiedene Operatoren:

```php
$F['ARTICLE']['W'][0]['Price']['>']  = 10;
$F['ARTICLE']['W'][0]['Price']['<']  = 100;
$F['ARTICLE']['W'][0]['Active']['!='] = 0;
```

Unterstützte Operatoren:

| Operator | Bedeutung |
|----------|-----------|
| `=` | Gleichheit |
| `!=` | Ungleich |
| `<>` | Ungleich (Alternative) |
| `>` | Größer als |
| `<` | Kleiner als |
| `>=` | Größer/gleich |
| `<=` | Kleiner/gleich |

---

# **4.5 LIKE‑Varianten**

CData bietet mehrere Varianten für LIKE‑Abfragen:

```php
$F['ARTICLE']['W'][0]['Name']['LIKE']   = '%test%';   // enthält
$F['ARTICLE']['W'][0]['Name']['LIKE%%'] = 'test';     // enthält (Kurzform)
$F['ARTICLE']['W'][0]['Name']['LIKE%-'] = 'test';     // endet mit
$F['ARTICLE']['W'][0]['Name']['LIKE-%'] = 'test';     // beginnt mit
```

| Syntax | Bedeutung |
|--------|-----------|
| LIKE | `%wert%` |
| LIKE%% | `%wert%` |
| LIKE%- | `%wert` |
| LIKE-% | `wert%` |

---

# **4.6 IN / NOT IN**

### IN‑Abfrage (Kurzform)

```php
$F['ARTICLE']['W'][0]['ID'] = ['1','2','3'];
```

### NOT IN

```php
$F['ARTICLE']['W'][0]['ID']['NOTIN'] = ['1','2','3'];
```

---

# **4.7 Filter auf CHILD‑Tabellen**

CData erlaubt das Filtern über rekursive Strukturen.

### Artikel laden, die ein Attribut „Hallo“ besitzen

```php
$F['ARTICLE']['W'][0]['ATTRIBUTE']['W'][0]['Name'] = 'Hallo';
```

→ Artikel werden gefiltert\
→ Nur Artikel mit Attribut „Hallo“ werden geladen

---

### Alle Artikel laden, aber nur Attribute mit Name „Hallo“ anzeigen

```php
$F['ARTICLE']['ATTRIBUTE']['W'][0]['Name'] = 'Hallo';
```

→ Artikel werden **nicht** gefiltert\
→ Nur die CHILD‑Daten werden gefiltert

---

# **4.8 Kombination von Filtern**

Beispiel:\
Alle aktiven Artikel, die entweder „A“ oder „B“ heißen:

```php
$F['ARTICLE']['W'][0]['Active'] = 1;
$F['ARTICLE']['W'][1]['Name']   = 'A';
$F['ARTICLE']['W'][2]['Name']   = 'B';
```

---

# **4.9 Filter auf mehreren Ebenen**

Beispiel:\
Artikel → Attribute → Language

```php
$F['ARTICLE']['ATTRIBUTE']['LANGUAGE']['W'][0]['Title']['LIKE'] = '%rot%';
```

---

# **4.10 Filter + Sortierung + Limit kombinieren**

```php
$F['ARTICLE']['W'][0]['Active'] = 1;
$F['ARTICLE']['O'][0]['Name']   = 'ASC';
$F['ARTICLE']['L']['STEP']      = 20;
$F['ARTICLE']['L']['START']     = 0;
```

---

# **Zusammenfassung**

- Filter werden über `$F` definiert
- `W[0]` = AND, `W[1]` = OR
- Operatoren: `>`, `<`, `!=`, LIKE, IN, NOTIN
- CHILD‑Tabellen können gefiltert oder separat eingeschränkt werden
- Filter können beliebig tief verschachtelt werden
- Sortierung und Limit funktionieren auf jeder Ebene
