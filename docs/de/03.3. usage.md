# **6. Daten speichern – set_object`()`**

Die Methode `set_object()` ist das zentrale Werkzeug, um Daten in der Datenbank zu speichern, zu aktualisieren oder zu löschen.\
CData erkennt dabei **automatisch**, ob ein Datensatz neu angelegt, aktualisiert oder gelöscht werden soll – basierend auf der Struktur des `$D`‑Arrays.

```php
$db->set_object($D);
```

---

# **6.1 Grundprinzip**

Alle zu speichernden Daten werden in `$D` abgelegt:

```php
$D['TABELLE']['D'][ID][Feldname] = Wert;
```

Beispiel:

```php
$D['ARTICLE']['D']['id-artikel']['Active'] = 1;
$D['ARTICLE']['D']['id-artikel']['Name']   = 'Mein Artikel';
```

Beim Aufruf von:

```php
$db->set_object($D);
```

entscheidet CData automatisch:

- **Insert**, wenn die ID noch nicht existiert
- **Update**, wenn die ID bereits existiert
- **Delete**, wenn `__DELETE__` gesetzt ist

---

# **6.2 Insert – neuen Datensatz anlegen**

Ein neuer Datensatz wird angelegt, wenn die ID noch nicht existiert.

```php
$D['ARTICLE']['D']['A100']['Active'] = 1;
$D['ARTICLE']['D']['A100']['Name']   = 'Neuer Artikel';

$db->set_object($D);
```

CData erzeugt automatisch:

- neuen Datensatz
- alle CHILD‑Daten
- alle LANGUAGE‑Daten
- alle ATTRIBUTE‑Daten

---

# **6.3 Update – bestehenden Datensatz ändern**

Wenn die ID bereits existiert, wird ein Update ausgeführt.

```php
$D['ARTICLE']['D']['A100']['Name'] = 'Artikel aktualisiert';

$db->set_object($D);
```

Nur die übergebenen Felder werden geändert.

---

# **6.4 Delete – Datensatz löschen**

Ein Datensatz wird gelöscht, indem `__DELETE__` gesetzt wird:

```php
$D['ARTICLE']['D']['A100']['__DELETE__'] = 1;

$db->set_object($D);
```

CData löscht:

- den Datensatz
- alle CHILD‑Daten
- alle LANGUAGE‑Daten
- alle ATTRIBUTE‑Daten

---

# **6.5 CHILD‑Daten speichern**

CHILD‑Daten werden rekursiv gespeichert.

Beispiel: Artikel + Attribute

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['D']['Color']['Name'] = 'Rot';
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['D']['Size']['Name']  = 'XL';

$db->set_object($D);
```

CData speichert:

- ARTICLE
- ARTICLE → ATTRIBUTE

---

# **6.6 CHILD‑Daten löschen**

### Einzelnes CHILD‑Element löschen

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['Color']['__DELETE__'] = 1;
```

### Alle CHILD‑Elemente löschen

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['__DELETE__'] = 1;
```

---

# **6.7 LANGUAGE‑Daten speichern**

LANGUAGE ist keine spezielle Tabelle – sie wird wie jede andere CHILD‑Struktur behandelt.

```php
$D['ARTICLE']['D']['A100']['LANGUAGE']['DE']['Title'] = 'Testtitel';
$D['ARTICLE']['D']['A100']['LANGUAGE']['EN']['Title'] = 'Test title';

$db->set_object($D);
```

CData speichert automatisch:

- ARTICLE
- ARTICLE → LANGUAGE (DE, EN)

---

# **6.8 Mehrere Tabellen gleichzeitig speichern**

CData kann beliebig viele Tabellen in einem einzigen Durchlauf speichern.

```php
$D['ARTICLE']   = [...];
$D['ATTRIBUTE'] = [...];
$D['STOCK']     = [...];

$db->set_object($D);
```

CData verarbeitet:

- alle Tabellen
- alle CHILD‑Strukturen
- alle Inserts
- alle Updates
- alle Deletes

in einem einzigen rekursiven Vorgang.

---

# **6.9 Tiefe Strukturen speichern**

CData unterstützt unbegrenzte Tiefe:

```
ARTICLE
  ATTRIBUTE
    PROPERTY
      LANGUAGE
        META
```

Beispiel:

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['D']['Color']['PROPERTY']['D']['HEX']['Value'] = '#FF0000';

$db->set_object($D);
```

---

# **6.10 Best Practices**

- IDs sollten eindeutig und stabil sein
- CHILD‑Daten immer unter `D` ablegen
- `__DELETE__` nur setzen, wenn wirklich gelöscht werden soll
- `$D` kann beliebig viele Tabellen gleichzeitig enthalten
- Die Struktur von `$D` entspricht immer dem Pattern

---

# **Zusammenfassung**

- `set_object()` speichert Daten rekursiv anhand der Struktur von `$D`
- Insert, Update und Delete werden automatisch erkannt
- CHILD‑Tabellen, LANGUAGE und ATTRIBUTE werden vollständig unterstützt
- Mehrere Tabellen können gleichzeitig gespeichert werden
- Die Struktur kann beliebig tief sein
