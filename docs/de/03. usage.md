# **3. Daten lesen – get_object`()`**

Die Methode `get_object()` ist der zentrale Mechanismus, um Daten aus der Datenbank zu laden.\
Sie arbeitet ausschließlich mit dem **Filter‑Array `$F`** und schreibt die Ergebnisse in das **Daten‑Array `$D`**.\
Die Struktur von `$D` entspricht dabei exakt der Struktur, die auch beim Speichern (`set_object()`) verwendet wird.

---

# **Grundprinzip**

```php
$db->get_data($D, $F);
```

- `$F` bestimmt **was** geladen wird
- `$D` wird mit den geladenen Daten gefüllt
- Die Struktur ist rekursiv und folgt dem Pattern
- CHILD‑Tabellen werden automatisch mitgeladen, wenn sie im Filter angegeben sind

---

# **3.1 Tabellen laden**

Um eine Tabelle zu laden, reicht ein leerer Eintrag:

```php
$F['ARTICLE'] = [];
$db->get_object($D, $F);
```

Ergebnis:

```php
$D['ARTICLE']['D'][ID][Feldname]
```

---

# **3.2 Datensatz nach ID laden**

```php
$F['ARTICLE']['W'][0]['ID'] = '123';
```

Mehrere IDs:

```php
$F['ARTICLE']['W'][0]['ID'] = ['123', '456'];
```

---

# **3.3 WHERE‑Filter (W)**

### AND‑Filter (W\[0\])

```php
$F['ARTICLE']['W'][0]['Active'] = 1;
$F['ARTICLE']['W'][0]['Name']   = 'Artikel A';
```

### OR‑Filter (W\[1\])

```php
$F['ARTICLE']['W'][1]['Name'] = 'Artikel B';
```

---

# **3.4 Vergleichsoperatoren**

```php
$F['ARTICLE']['W'][0]['Price']['>']  = 10;
$F['ARTICLE']['W'][0]['Price']['<']  = 100;
$F['ARTICLE']['W'][0]['Active']['!='] = 0;
```

---

# **3.5 LIKE‑Varianten**

CData unterstützt mehrere LIKE‑Formen:

```php
$F['ARTICLE']['W'][0]['Name']['LIKE']   = '%test%';   // enthält
$F['ARTICLE']['W'][0]['Name']['LIKE%%'] = 'test';     // enthält (Kurzform)
$F['ARTICLE']['W'][0]['Name']['LIKE%-'] = 'test';     // endet mit
$F['ARTICLE']['W'][0]['Name']['LIKE-%'] = 'test';     // beginnt mit
```

---

# **3.6 IN / NOT IN**

```php
$F['ARTICLE']['W'][0]['ID'] = ['1','2','3'];          // IN
$F['ARTICLE']['W'][0]['ID']['NOTIN'] = ['1','2','3']; // NOT IN
```

---

# **3.7 Sortierung – ORDER (O)**

```php
$F['ARTICLE']['O'][0]['Name'] = 'ASC';
```

Mehrere Sortierungen:

```php
$F['ARTICLE']['O'][0]['Active'] = 'DESC';
$F['ARTICLE']['O'][1]['Name']   = 'ASC';
```

---

# **3.8 Limit & Offset – LIMIT (L)**

```php
$F['ARTICLE']['L']['STEP']  = 10; // Anzahl
$F['ARTICLE']['L']['START'] = 0;  // Offset
```

---

# **3.9 CHILD‑Tabellen laden**

CHILD‑Tabellen werden über das Pattern erkannt und können gezielt geladen werden.

### Alle Artikel + alle Attribute

```php
$F['ARTICLE']['ATTRIBUTE'] = [];
```

### Artikel filtern nach Attributen

```php
$F['ARTICLE']['W'][0]['ATTRIBUTE']['W'][0]['Name'] = 'Hallo';
```

→ Nur Artikel, die ein Attribut „Hallo“ besitzen

### Alle Artikel, aber nur bestimmte Attribute laden

```php
$F['ARTICLE']['ATTRIBUTE']['W'][0]['Name'] = 'Hallo';
```

→ Artikel werden **nicht** gefiltert, nur die Attribute

---

# **3.10 Mehrere Tabellen gleichzeitig laden**

```php
$F['ARTICLE']  = [];
$F['ATTRIBUTE'] = [];
$F['STOCK']     = [];

$db->get_object($D, $F);
```

---

# **3.11 Ergebnisstruktur in `$D`**

Nach dem Laden sieht `$D` z. B. so aus:

```php
$D['ARTICLE']['D']['123']['Active'] = 1;
$D['ARTICLE']['D']['123']['Name']   = 'Testartikel';

$D['ARTICLE']['D']['123']['ATTRIBUTE']['D']['Color']['Name'] = 'Rot';
```

Die Struktur entspricht exakt dem Pattern.

---

# **Zusammenfassung**

- `get_object()` lädt Daten rekursiv anhand des Filters `$F`
- `$D` enthält die vollständige Datenstruktur
- Filter unterstützen AND, OR, LIKE, IN, NOT IN, Operatoren
- CHILD‑Tabellen können gefiltert oder separat geladen werden
- Sortierung und Limit funktionieren auf jeder Ebene
- Die Struktur ist beliebig tief
