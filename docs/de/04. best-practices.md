# **8. Best Practices**

Dieses Kapitel fasst bewährte Vorgehensweisen zusammen, die dir helfen, CData effizient, stabil und sauber zu nutzen.\
Die folgenden Empfehlungen basieren auf typischen Anwendungsfällen, häufigen Fehlerquellen und optimalen Strukturierungsstrategien für komplexe Datenmodelle.

---

# **8.1 Saubere und stabile IDs verwenden**

CData verwendet **immer den Array‑Key als ID** eines Datensatzes:

```php
$D['ARTICLE']['D']['A100']
```

Best Practices:

- IDs sollten **eindeutig** sein
- IDs sollten **nicht dynamisch wechseln**
- IDs dürfen **alphanumerisch** sein
- IDs sollten **keine Leerzeichen** enthalten

Gute Beispiele:

```
A100
ART-001
COLOR_RED
```

Schlechte Beispiele:

```
123 456
Artikel 1
```

---

# **8.2 Pattern klar und vollständig definieren**

Das Pattern ist das Fundament von CData.\
Achte darauf:

- Jede Tabelle sollte ein vollständiges Pattern besitzen
- CHILD‑Tabellen immer unter `D` definieren
- Datentypen sinnvoll wählen
- Foreign Keys nur setzen, wenn sie wirklich benötigt werden

Beispiel:

```php
$D['PATTERN']['ARTICLE'] = [
    'Active' => ['Type' => 'checkbox'],
    'Name'   => ['Type' => 'text'],
];
```

---

# **8.3 `$D` und `$F` strikt trennen**

- `$D` = Datencontainer (Lesen & Schreiben)
- `$F` = Filtercontainer (Abfragen)

Vermische diese beiden Strukturen niemals.

---

# **8.4 Nur benötigte Daten laden**

CData kann sehr große Strukturen laden.\
Lade nur das, was du wirklich brauchst:

```php
$F['ARTICLE'] = []; // lädt nur Artikel
```

Oder:

```php
$F['ARTICLE']['ATTRIBUTE'] = []; // lädt Artikel + Attribute
```

Vermeide:

```php
$F['ARTICLE']['ATTRIBUTE']['PROPERTY']['LANGUAGE']['META'] = [];
```

wenn du nur Artikel brauchst.

---

# **8.5 CHILD‑Filter bewusst einsetzen**

Unterschied:

### Artikel filtern nach Attributen

```php
$F['ARTICLE']['W'][0]['ATTRIBUTE']['W'][0]['Name'] = 'Rot';
```

→ Nur Artikel mit Attribut „Rot“

### Artikel laden, aber Attribute filtern

```php
$F['ARTICLE']['ATTRIBUTE']['W'][0]['Name'] = 'Rot';
```

→ Alle Artikel, aber nur Attribute „Rot“

Das ist ein häufiger Fehler – achte auf die Position des Filters.

---

# **8.6 `__DELETE__` sparsam und gezielt einsetzen**

Ein Datensatz löschen:

```php
$D['ARTICLE']['D']['A100']['__DELETE__'] = 1;
```

Ein einzelnes CHILD‑Element löschen:

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['Color']['__DELETE__'] = 1;
```

Alle CHILD‑Elemente löschen:

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['__DELETE__'] = 1;
```

Achte darauf:

- Keine unnötigen Löschmarkierungen setzen
- CHILD‑Löschungen bewusst platzieren

---

# **8.7 Tiefe Strukturen sinnvoll planen**

CData unterstützt unbegrenzte Tiefe – aber:

- Plane die Struktur vorher
- Vermeide unnötig tiefe Verschachtelungen
- Nutze CHILD‑Tabellen nur, wenn sie logisch notwendig sind
- Halte die Struktur konsistent

Beispiel für gute Struktur:

```
ARTICLE
  ATTRIBUTE
    LANGUAGE
```

Beispiel für schlechte Struktur:

```
ARTICLE
  ATTRIBUTE
    PROPERTY
      META
        EXTRA
          MORE
            AGAIN
```

---

# **8.8 Filter, Sortierung und Limit kombinieren**

Ein typisches Beispiel:

```php
$F['ARTICLE']['W'][0]['Active'] = 1;
$F['ARTICLE']['O'][0]['Name']   = 'ASC';
$F['ARTICLE']['L']['STEP']      = 20;
$F['ARTICLE']['L']['START']     = 0;
```

Best Practices:

- Sortierung immer vor Limit setzen
- Filter so spezifisch wie möglich halten
- CHILD‑Sortierung nur anwenden, wenn nötig

---

# **8.9 Mehrere Tabellen gleichzeitig speichern**

CData kann mehrere Tabellen in einem Durchlauf speichern:

```php
$D['ARTICLE']   = [...];
$D['ATTRIBUTE'] = [...];
$D['STOCK']     = [...];

$db->set_object($D);
```

Best Practices:

- `$D` klar strukturieren
- Keine unnötigen Tabellen einfügen
- CHILD‑Daten immer unter `D` ablegen

---

# **8.10 Debugging‑Tipps**

- `$D` nach `get_object()` immer prüfen
- `$F` vor dem Laden ausgeben
- Pattern auf Vollständigkeit prüfen
- IDs prüfen, wenn Daten nicht gespeichert werden
- `__DELETE__` prüfen, wenn Daten verschwinden

---

# **Zusammenfassung**

- IDs stabil und eindeutig halten
- Pattern sauber definieren
- `$D` und `$F` strikt trennen
- Nur benötigte Daten laden
- CHILD‑Filter korrekt einsetzen
- Löschmarkierungen bewusst verwenden
- Tiefe Strukturen sinnvoll planen
- Filter, Sortierung und Limit kombinieren
- Mehrere Tabellen sauber gleichzeitig speichern
- `$D` und `$F` regelmäßig prüfen

Diese Best Practices helfen dir, CData effizient, stabil und fehlerfrei einzusetzen.
