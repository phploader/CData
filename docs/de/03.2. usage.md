# **5. Sortierung & Limit**

CData bietet eine flexible Möglichkeit, Daten zu sortieren und die Anzahl der zurückgegebenen Datensätze zu begrenzen.\
Diese Funktionen werden vollständig über das Filter‑Array **`$F`** gesteuert und können auf jeder Ebene einer Datenstruktur angewendet werden – auch auf CHILD‑Tabellen.

---

# **5.1 Sortierung – ORDER (O)**

Sortierungen werden über den Schlüssel **`O`** definiert.\
Jeder Sortierblock wird nummeriert (`O[0]`, `O[1]`, …) und entspricht einer Sortierpriorität.

Beispiel: Nach Name aufsteigend sortieren

```php
$F['ARTICLE']['O'][0]['Name'] = 'ASC';
```

Beispiel: Erst nach Active (DESC), dann nach Name (ASC)

```php
$F['ARTICLE']['O'][0]['Active'] = 'DESC';
$F['ARTICLE']['O'][1]['Name']   = 'ASC';
```

### Unterstützte Sortierrichtungen

| Wert | Bedeutung |
|------|-----------|
| `ASC` | Aufsteigend |
| `DESC` | Absteigend |

---

# **5.2 Sortierung auf CHILD‑Tabellen**

Sortierung kann auch auf CHILD‑Daten angewendet werden.

Beispiel: Artikel laden, Attribute alphabetisch sortieren

```php
$F['ARTICLE']['ATTRIBUTE']['O'][0]['Name'] = 'ASC';
```

Beispiel: Artikel laden, Attribute nach Active sortieren

```php
$F['ARTICLE']['ATTRIBUTE']['O'][0]['Active'] = 'DESC';
```

---

# **5.3 Limit – Anzahl der Datensätze begrenzen**

CData verwendet für LIMIT zwei Parameter:

- **`STEP`** → Anzahl der Datensätze
- **`START`** → Offset (Startposition)

Beispiel: Die ersten 10 Artikel laden

```php
$F['ARTICLE']['L']['STEP']  = 10;
$F['ARTICLE']['L']['START'] = 0;
```

Beispiel: Artikel 20–29 laden (Paging)

```php
$F['ARTICLE']['L']['STEP']  = 10;
$F['ARTICLE']['L']['START'] = 20;
```

---

# **5.4 Limit auf CHILD‑Tabellen**

Limit kann auch auf CHILD‑Daten angewendet werden.

Beispiel: Pro Artikel nur die ersten 5 Attribute laden

```php
$F['ARTICLE']['ATTRIBUTE']['L']['STEP']  = 5;
$F['ARTICLE']['ATTRIBUTE']['L']['START'] = 0;
```

---

# **5.5 Kombination von Sortierung & Limit**

Sortierung und Limit können beliebig kombiniert werden.

Beispiel:\
Alle aktiven Artikel, sortiert nach Name, aber nur die ersten 20:

```php
$F['ARTICLE']['W'][0]['Active'] = 1;
$F['ARTICLE']['O'][0]['Name']   = 'ASC';
$F['ARTICLE']['L']['STEP']      = 20;
$F['ARTICLE']['L']['START']     = 0;
```

---

# **5.6 Sortierung & Limit in tiefen Strukturen**

CData unterstützt Sortierung und Limit in beliebig tiefen CHILD‑Strukturen.

Beispiel:

```php
$F['ARTICLE']['ATTRIBUTE']['LANGUAGE']['O'][0]['Title'] = 'ASC';
$F['ARTICLE']['ATTRIBUTE']['LANGUAGE']['L']['STEP'] = 10;
```

---

# **Zusammenfassung**

- Sortierung erfolgt über `O` (ORDER)
- Limit erfolgt über `L` (STEP & START)
- Beide Funktionen können auf jeder Ebene angewendet werden
- Mehrere Sortierregeln sind möglich
- Sortierung & Limit funktionieren auch auf CHILD‑Tabellen
- Ideal für Paging, alphabetische Listen, Produktkataloge usw.
