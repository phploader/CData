# **7. Rekursive Datenmodelle**

CData wurde von Grund auf dafür entwickelt, **rekursive und verschachtelte Datenstrukturen beliebiger Tiefe** zu unterstützen.\
Im Gegensatz zu klassischen ORMs oder SQL‑Modellen gibt es in CData **keine feste Begrenzung**, wie tief eine Struktur verschachtelt sein darf.\
Alles wird über das Pattern definiert – und CData übernimmt automatisch die Speicherung, das Laden und die Verknüpfung der Daten.

---

# **7.1 Was bedeutet „rekursives Datenmodell“?**

Ein rekursives Datenmodell ist eine Struktur, bei der Datensätze:

- Unterelemente besitzen
- diese Unterelemente wiederum Unterelemente besitzen
- und so weiter – theoretisch unendlich tief

Beispiel:

```
ARTICLE
  ATTRIBUTE
    PROPERTY
      LANGUAGE
        META
```

Jede Ebene kann eigene Felder, eigene CHILD‑Tabellen und eigene Strukturen besitzen.

---

# **7.2 Definition rekursiver Strukturen im Pattern**

Rekursion entsteht durch das Verwenden des Schlüssels **`D`** im Pattern.

Beispiel:

```php
$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE'] = [
    'Active' => ['Type' => 'checkbox'],
    'Name'   => ['Type' => 'text'],
];
```

Weitere Ebene:

```php
$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE']['D']['LANGUAGE'] = [
    'Title' => ['Type' => 'text'],
];
```

Noch eine Ebene:

```php
$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE']['D']['LANGUAGE']['D']['META'] = [
    'Info' => ['Type' => 'text'],
];
```

CData verarbeitet diese Struktur vollständig automatisch.

---

# **7.3 Speicherung rekursiver Daten**

Rekursive Daten werden einfach in `$D` abgelegt:

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['D']['Color']['PROPERTY']['D']['HEX']['Value'] = '#FF0000';
```

Beim Speichern:

```php
$db->set_object($D);
```

CData erkennt:

- welche Ebene neu ist
- welche Ebene aktualisiert wird
- welche Ebene gelöscht werden soll
- wie die Beziehungen zwischen den Ebenen aussehen

---

# **7.4 Laden rekursiver Daten**

Rekursive Daten werden über `$F` geladen.

Beispiel: Artikel + Attribute + Language

```php
$F['ARTICLE']['ATTRIBUTE']['LANGUAGE'] = [];
```

Beispiel: Artikel + Attribute + Language + Meta

```php
$F['ARTICLE']['ATTRIBUTE']['LANGUAGE']['META'] = [];
```

CData baut automatisch die komplette Struktur in `$D` auf:

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['D']['Color']['LANGUAGE']['D']['DE']['META']['D']['Info']
```

---

# **7.5 Filtern in rekursiven Strukturen**

Filter können auf jeder Ebene angewendet werden.

Beispiel:\
Artikel laden, deren Attribute eine Sprache „DE“ besitzen:

```php
$F['ARTICLE']['ATTRIBUTE']['LANGUAGE']['W'][0]['ID'] = 'DE';
```

Beispiel:\
Artikel laden, deren Attribute eine bestimmte Meta‑Information enthalten:

```php
$F['ARTICLE']['ATTRIBUTE']['LANGUAGE']['META']['W'][0]['Info']['LIKE'] = '%rot%';
```

---

# **7.6 Sortierung & Limit in rekursiven Strukturen**

Sortierung:

```php
$F['ARTICLE']['ATTRIBUTE']['LANGUAGE']['O'][0]['Title'] = 'ASC';
```

Limit:

```php
$F['ARTICLE']['ATTRIBUTE']['LANGUAGE']['L']['STEP'] = 10;
```

Beides funktioniert auf jeder Ebene.

---

# **7.7 Tiefe ist unbegrenzt**

CData hat **keine Begrenzung**, wie tief ein rekursives Modell verschachtelt sein darf.\
Die einzige Grenze ist die verfügbare Systemressource.

Das bedeutet:

- Du kannst beliebig komplexe Datenmodelle abbilden
- Jede Ebene kann eigene CHILD‑Tabellen besitzen
- Jede Ebene kann eigene Filter, Sortierungen und Limits haben
- Die Struktur bleibt vollständig konsistent

---

# **7.8 Beispiel: Vollständiges rekursives Modell**

Pattern:

```php
$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE']['D']['PROPERTY']['D']['LANGUAGE']['D']['META'] = [
    'Info' => ['Type' => 'text'],
];
```

Speichern:

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['D']['Color']['PROPERTY']['D']['HEX']['LANGUAGE']['D']['DE']['META']['D']['M1']['Info'] = 'Rot';
```

Laden:

```php
$F['ARTICLE']['ATTRIBUTE']['PROPERTY']['LANGUAGE']['META'] = [];
```

Ergebnis:

```php
$D['ARTICLE']['D']['A100']['ATTRIBUTE']['D']['Color']['PROPERTY']['D']['HEX']['LANGUAGE']['D']['DE']['META']['D']['M1']['Info']
```

---

# **Zusammenfassung**

- Rekursive Datenmodelle sind ein Kernfeature von CData
- Die Tiefe der Struktur ist unbegrenzt
- CHILD‑Tabellen werden über `D` definiert
- Speicherung und Laden erfolgen automatisch rekursiv
- Filter, Sortierung und Limit funktionieren auf jeder Ebene
- Komplexe Modelle wie Artikel → Attribute → Property → Language → Meta sind problemlos möglich
