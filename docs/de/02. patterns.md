# **2. Pattern‑Definition**

Das Pattern‑System ist das Fundament von CData.\
Es beschreibt die komplette Datenbankstruktur – Tabellen, Felder, Datentypen, Beziehungen und rekursive CHILD‑Strukturen.\
CData erzeugt anhand des Pattern automatisch die SQLite‑Tabellen und verwaltet alle Datenoperationen (Insert, Update, Delete).

Ein Pattern wird einmalig definiert und anschließend mit:

```php
$C['CData']->registerPattern($D['PATTERN']);
```

registriert.

---

# **Was ist ein Pattern?**

Ein Pattern ist ein Array, das beschreibt:

- welche Tabellen existieren
- welche Felder diese Tabellen haben
- welche Datentypen die Felder besitzen
- welche CHILD‑Tabellen dazugehören
- wie tief die Struktur geht (beliebig tief)

Beispiel für eine einfache Tabelle:

```php
$D['PATTERN']['ARTICLE'] = [
    'Active'   => ['Type' => 'checkbox'],
    'Name'     => ['Type' => 'text'],
    'ParentId' => ['Type' => 'id', 'ForeignKey' => 1],
];
```

Damit erzeugt CData automatisch eine Tabelle `ARTICLE` mit den Feldern:

- Active
- Name
- ParentId

---

# **Unterstützte Datentypen**

| Type | Beschreibung |
|------|--------------|
| checkbox | boolescher Wert (0/1) |
| text | normaler Text |
| id | alphanumerische ID |
| number | numerischer Wert |
| json | JSON‑Wert |
| date | Datum |

> Neue Typen können jederzeit ergänzt werden, indem die CData‑Klasse erweitert wird.

---

# **CHILD‑Tabellen definieren**

CHILD‑Tabellen werden über den Schlüssel `D` definiert.\
Dies ermöglicht rekursive Strukturen beliebiger Tiefe.

Beispiel: Artikel mit Attributen

```php
$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE'] = [
    'Active' => ['Type' => 'checkbox'],
    'Name'   => ['Type' => 'text'],
];
```

Damit entsteht eine Struktur:

```
ARTICLE
  ATTRIBUTE
```

CData erzeugt automatisch die notwendigen Tabellen und Verknüpfungen.

---

# **Mehrstufige CHILD‑Strukturen**

CData unterstützt beliebig tiefe Strukturen.

Beispiel:

```php
$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE']['D']['LANGUAGE'] = [
    'Title' => ['Type' => 'text'],
];
```

Struktur:

```
ARTICLE
  ATTRIBUTE
    LANGUAGE
```

Diese Struktur kann beliebig erweitert werden:

```
ARTICLE
  ATTRIBUTE
    PROPERTY
      LANGUAGE
        META
```

Es gibt **keine Tiefenbegrenzung**.

---

# **Foreign Keys**

Ein Feld kann als Foreign Key markiert werden:

```php
'ParentId' => ['Type' => 'id', 'ForeignKey' => 1],
```

Dies dient dazu, Beziehungen abzubilden.\
CData erzeugt intern eine passende Struktur, z. B.:

```
$D['ARTICLE']['ForeignKey']['Kind-ids']
```

Foreign Keys werden **nicht** über SQL‑Constraints definiert, sondern über die interne Strukturverwaltung von CData.

---

# **Beispiel: Komplexes Pattern**

```php
$D['PATTERN']['ARTICLE'] = [
    'Active'   => ['Type' => 'checkbox'],
    'Name'     => ['Type' => 'text'],
    'ParentId' => ['Type' => 'id', 'ForeignKey' => 1],
];

$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE'] = [
    'Active' => ['Type' => 'checkbox'],
    'Name'   => ['Type' => 'text'],
];

$D['PATTERN']['ARTICLE']['D']['ATTRIBUTE']['D']['LANGUAGE'] = [
    'Title' => ['Type' => 'text'],
];
```

Ergebnisstruktur:

```
ARTICLE
  ATTRIBUTE
    LANGUAGE
```

---

# **Pattern registrieren**

Nach der Definition muss das Pattern registriert werden:

```php
$C['CData']->registerPattern($D['PATTERN']);
```

CData erzeugt daraufhin automatisch:

- Tabellen
- Felder
- CHILD‑Tabellen
- interne Beziehungen
- Speicherlogik

---

# **Zusammenfassung**

- Ein Pattern definiert die komplette Datenbankstruktur.
- Tabellen und Felder werden automatisch erzeugt.
- CHILD‑Tabellen werden über `D` rekursiv definiert.
- Die Struktur kann beliebig tief sein.
- Datentypen steuern die Speicherung und Verarbeitung.
- Foreign Keys dienen der internen Strukturverwaltung.

Das Pattern ist die Grundlage für alle Operationen in CData und bestimmt, wie Daten gelesen, gespeichert und strukturiert werden.
